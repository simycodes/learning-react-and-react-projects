A USER FROM THE FRONT END WHEN SENDING REQUESTING TO THE BACKEND MUST BE AUTHENTICATED.

JWT HELPS US AUTHENTICATE A USER, BY MAKING THE USER TO SEND THEIR REQUEST WITH AN Authorization HEADER THAT IS
THEN CHECKED ON THE BACKEND.

WITHOUT THIS JWT Authorization THEN THE USER WILL NOT BE ALLOWED TO ACCESS CERTAIN API ROUTES.


JWT Authorization HEADER EXAMPLE

KEY			VALUE

Authorization   Bearer <JWT>
